# Code Quality Tools

These are the commands to install ESLint, AirBnB styles + prettier components in our app:

~~~bash
npm install -D eslint
npm install -D eslint-config-airbnb-base
npm install -D eslint-config-prettier
npm install -D eslint-plugin-import
npm install -D prettier
npm install -D nodemon
~~~

The above will install these packages to the `node_modules` folder, and extend `package.json` as follows:

The `-D` option signifies that these are `developer` components, listed in a `devDependencies` section as shown.

Before proceeding any further, we need to reset eslint to an earlier version. Change this now:

~~~
    "eslint": "^8.57.0",
~~~

The latest major version - 9.0.0, is a little ahead of the eslint plugins at the moment, so we reset to version 8 until these catch up.

ESLint is configured by an additional file in your project. Create this now:

## .eslintrc.json

~~~json
{
  "env": {
    "node": true,
    "es2021": true
  },
  "extends": ["airbnb-base", "prettier"],
  "parserOptions": {
    "ecmaVersion": "latest",
    "sourceType": "module"
  },
  "rules": {
    "quotes": ["error", "double"],
    "import/extensions": "off",
    "import/prefer-default-export": "off",
    "object-shorthand": "off",
    "no-unused-vars": "off",
    "no-underscore-dangle": "off",
    "no-param-reassign": "off",
    "no-undef": "off",
    "import/no-unresolved": "off",
    "import/no-extraneous-dependencies": "off",
    "no-console": "off",
    "func-names": "off"
  }
}
~~~

In the above we customised some of the rules inherited from the airbnb style guide - tuning them to the specific needs/preferences of our application.

We configure prettier with this file:

## .prettierrc.json

~~~json
{
  "trailingComma": "es5",
  "tabWidth": 2,
  "semi": true,
  "singleQuote": false,
  "printWidth": 180
}
~~~
